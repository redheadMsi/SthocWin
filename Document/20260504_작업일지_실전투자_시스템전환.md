# 작업 일지: SthocWin 실전 투자 시스템 전환 및 고도화

**날짜:** 2026년 05월 04일
**목표:** 단순 백테스트 도구에서 API를 연동한 실시간 자동 매매 및 모니터링 시스템(Live Trading System)으로 격상

## 1. 오늘 작업 요약

### 1) 실전 투자 파이프라인(Broker) 구축
*   **환경 변수 관리:** `.env` 및 `config.py`를 도입하여 한국투자증권 API 키, 계좌번호 등 민감 정보를 분리.
*   **KIS API 모듈 (`broker_kis.py`):** 한국투자증권 Open API를 통해 토큰 발급, 해시키 암호화, 계좌 잔고 확인(`inquire-balance`), 매수/매도 주문 전송(`order-cash`)을 수행하는 객체지향 클래스 구현.
*   **안전망(Virtual Mode) 지원:** 환경 변수 하나(`KIS_MODE=VIRTUAL`)로 실전과 모의 투자를 스위칭하는 안전 장치 탑재.

### 2) 다중 채널 알림 라우터 구축 (`notifier.py`)
*   기존 디스코드 전용 발송 로직을 **다중 채널 라우터(Multi-channel Router) 아키텍처**로 개편.
*   `.env`의 `NOTIFICATION_CHANNELS` 설정(예: `DISCORD,SLACK,LINE`)에 따라 슬랙, 라인, 카카오톡, 디스코드 등 원하는 메신저로 알림을 동시에 뿌려주는 범용 `send_notification()` 함수 구현.
*   환경변수 세팅 누락 시 앱이 죽지 않고 Streamlit UI 화면 상단에 안전하게 [모의 알림] 박스를 띄워주도록 Graceful degradation 적용.

### 3) 종가 매매 엔진 (`bot_engine.py`) 및 일일 리포트 구현
*   **`run_daily_bot()`:** 매일 15:20에 봇이 깨어나 유니버스를 필터링하고 당일 주가를 연산하여 "살 종목, 팔 종목"을 색출. `AUTO_TRADING=True` 설정 시 즉각 한국투자증권 서버로 주문 쏘는 로직 적용.
*   **`run_daily_report()`:** 계좌의 총 자산, 예수금, 주식 평가금액 및 보유 종목의 개별 수익률을 가져와 보기 좋은 마크다운 리포트 형식으로 포맷팅하여 메신저로 전송.

### 4) 프론트엔드 대시보드 (`app.py`) 확장
*   상단 탭(`st.tabs`) 구조를 도입하여 "📊 백테스트"와 "🤖 실전 투자 봇" UI 분리.
*   실전 투자 탭에서 봇의 유니버스와 사용할 지표(MA, RSI, MACD 등)를 백테스트와 동일하게 체크박스로 직관적으로 제어 가능하도록 UI 통일성 확보.
*   원클릭으로 `.env` 파일을 복사/생성해주는 자동화 버튼 및 수동 시그널 테스트/리포트 발송 버튼 배치.

## 2. 리뷰 및 평가
과거 단순히 과거 데이터를 돌려보는 툴을 넘어서, "시장 분석 -> 시그널 탐지 -> 자동 매매 주문 -> 성과 리포팅"으로 이어지는 **완전한 퀀트 트레이딩의 생애주기(Lifecycle)**를 단일 프로젝트 안에 담아냈다는 점에서 의미 있는 마일스톤 달성입니다.
