## Problem Statement
기존 HTS/MTS에 의존하지 않고, 개인화된 백테스트와 실시간 자동 매매, 실시간 모니터링이 모두 가능한 독립적인 투자 환경을 구축하고자 함. 그 첫 마일스톤으로 "현재 거래량 상위 10개 종목에 대한 2년치 백테스트 결과를 웹 대시보드로 실시간 확인"하는 것을 목표로 함.

## What Makes This Cool
단순히 터미널에 텍스트를 출력하는 스크립트가 아니라, **Streamlit**을 이용해 웹 브라우저 상에서 동작하는 인터랙티브 대시보드를 구축. 이동평균선 일수(5일, 20일 등)를 슬라이더로 조절하면 백테스트 결과 차트와 수익률이 즉각적으로 변하는 살아있는 앱 형태가 됨. 

## Constraints
- 시장: 한국 주식 시장 (코스피/코스닥)
- 개발 언어 및 도구: Python, pandas (데이터 처리), pykrx 또는 FinanceDataReader (데이터 수집)
- 무거운 기존 퀀트 프레임워크(Backtrader 등)는 피하고, 직관적인 커스텀 로직 유지

## Premises
1. **대상 시장:** 한국 주식 시장(코스피/코스닥) 기준. 당일 거래량 상위 10개 종목을 추출하고 2년 치 일봉 데이터를 가져온다.
2. **테스트 전략:** 5일/20일 이동평균선 교차 전략(골든크로스 매수, 데드크로스 매도)을 베이스라인으로 삼는다.
3. **사용 도구:** 시그널 계산은 `pandas`로 직접 통제하고, 시각화와 웹 UI는 `Streamlit`과 `vectorbt`로 구축한다.

## Cross-Model Perspective
SECOND OPINION (Claude subagent):
1. 가장 멋진 10x 버전: Streamlit 대시보드에 슬라이더를 붙여 파라미터를 즉각 조절하고, Github Actions/디스코드로 매일 시그널을 쏴주는 봇 결합.
2. 50/50 스플릿: 통계 연산과 차트는 `vectorbt`로 50%를 날로 먹고, 나머지 50%인 '거래량 상위 스크래핑 + 시그널 전처리 파이프라인'에 집중할 것.
3. 가장 먼저 할 일: 한 번에 10개 종목을 돌리지 말고, 단일 종목(예: 삼성전자)에 대한 수직 관통 파이프라인을 가장 먼저 뚫을 것.

## Approaches Considered
### Approach A: 순수 Pandas 수동 구현 (The Minimal Viable)
  Summary: 외부 프레임워크 없이 오직 pandas만 이용해서 매매 로직과 누적 수익률 계산
  Effort:  Medium
  Risk:    Low
  Pros:    - 바닥부터 내부 수학 완벽 통제 가능
  Cons:    - 승률, MDD 등 통계 지표와 시각화를 바닥부터 짜야 해서 코드가 매우 길어짐

### Approach B: Pandas + Vectorbt 결합 (The Ideal Architecture)
  Summary: 시그널 계산은 pandas로 통제하고, 수익률/차트 연산은 vectorbt에 위임
  Effort:  Small
  Risk:    Low
  Pros:    - 코드량이 1/3로 줄어들고 전문 퀀트 툴 수준의 차트를 2줄만에 획득
  Cons:    - vectorbt 라이브러리 추가 학습 필요

### Approach C: Streamlit 대시보드 + Vectorbt (The Lateral/Creative) - **[선택됨]**
  Summary: B안의 코어 엔진 위에 Streamlit 웹 프론트엔드를 덮어씌워 인터랙티브 대시보드로 구동
  Effort:  Large
  Risk:    Medium
  Pros:    - 당장 공유하고 사용할 수 있는 완성된 '제품' 형태의 앱 산출
  Cons:    - UI 구성 작업이 추가되므로 첫 주말에 소화하기 벅찰 수 있음

## Recommended Approach
**APPROACH C (Streamlit 대시보드)** 채택.
추천(B안)을 넘어 C안을 선택함으로써, "단순 스크립트"가 아닌 진짜 "사용 가능한 제품"을 만들겠다는 강한 의지(Agency)를 보여줌.

## Success Criteria
1. 코드를 실행하면 로컬 웹 브라우저에 대시보드가 뜬다 (`streamlit run app.py`).
2. 대시보드 진입 시 한국 시장 거래량 상위 10개 종목의 리스트가 자동으로 불러와진다.
3. 종목을 선택하면 과거 2년 치 데이터 기반의 5/20 이평선 백테스트 수익률 차트와 승률/MDD가 화면에 출력된다.

## Distribution Plan
로컬 환경 구동. 향후 Github에 올리고 Streamlit Community Cloud를 통해 무료로 웹 호스팅하여 개인 도메인으로 접속 가능하도록 확장.

## Next Steps
1. **Core 엔진 구현:** `pykrx` 또는 `FinanceDataReader`로 거래량 상위 종목 리스트 추출 및 단일 종목 주가 데이터 확보 함수 작성.
2. **시그널 처리:** `pandas`로 5/20일선 시그널(True/False) 생성 코드 작성.
3. **UI 씌우기:** `streamlit`을 이용해 종목 선택 드롭다운과 백테스트 차트를 출력하는 웹 페이지 구성.
