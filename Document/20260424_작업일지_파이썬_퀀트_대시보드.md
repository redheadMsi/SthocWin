# 작업 일지: 파이썬 퀀트 투자 대시보드 (통합 시뮬레이터)

**날짜:** 2026년 04월 24일
**프로젝트:** SthocWin (파이썬 퀀트 투자 대시보드)
**목표:** 우량주 대상 다중 파라미터 백테스트 결과를 웹 대시보드로 비교 분석 (첫 마일스톤)

## 1. 오늘 작업 요약
*   **초기 UI 프레임워크 구축 (app.py):**
    *   Streamlit을 사용하여 백테스트 시뮬레이터의 뼈대(레이아웃)를 작성.
    *   유니버스 선택 드롭다운, 다중 파라미터 입력(`5/20, 10/60...` 형태), 실행 버튼 구성.
    *   UI 레이아웃이 정상적으로 나타나는지 가짜(Dummy) 데이터로 최초 검증 완료.

*   **Core 연산 엔진 연결 (core_engine.py):**
    *   `FinanceDataReader`를 이용하여 'KOSPI 200 / KOSDAQ 150' 또는 '당일 시총 상위 100개'에서 '거래대금 상위 10개 종목'을 필터링하는 로직 구현.
    *   각 종목에 대한 지정 기간 내 이동평균선(단기, 장기) 교차 전략의 실 수익률 및 승률을 연산하는 기능 구현.
    *   최초 `pykrx` 라이브러리로 KOSPI 200 구성 종목을 호출 시 발생한 에러(`IndexError`)를 해결하기 위해, `FinanceDataReader` 단일 구조로 스크래핑 엔진 완전 교체.
    *   실행 시마다 결괏값이 조금씩 달라지는 문제 수정을 위해, 연산 기준 날짜의 시간을 자정(0시 0분 0초)으로 고정하여 일관성 확보.

*   **UX 업그레이드 (차트 및 기간 선택 기능 추가):**
    *   **기간 지정 기능:** 단순히 2년 전 데이터만 가져오던 로직에서 발전하여, 사용자가 **시작일과 종료일 달력(Date Input)**에서 직접 백테스트 기간을 선택할 수 있도록 수정.
    *   **인터랙티브 수익률 비교:** 종목별 테스트 결과를 `Plotly` 라이브러리의 막대 차트(Bar chart)로 시각화 (수익/손실 색상 자동 구분).
    *   **상세 주가/이평선 차트 렌더링:** 위 막대 차트에서 특정 종목을 **클릭(Select)**하면, 해당 종목의 지정 기간 주가 캔들(라인)과 이동평균선 2개가 함께 겹쳐져 상세 꺾은선 차트(Line chart)로 즉시 렌더링되는 기능 완성. (이 과정에서 Streamlit의 `st.session_state`를 이용해 화면 리로딩 시 결과 초기화 방지)

## 2. 발생했던 주요 이슈 및 해결
1.  **초기 백색 화면:** 경로 문제로 `app.py` 파일이 잘못 생성되어 빈 파일이 실행되는 이슈 -> 파일 경로를 맞추어 정상 복구.
2.  **pykrx KOSPI 200 조회 오류:** `IndexError: index -1 is out of bounds for axis 0 with size 0` -> FinanceDataReader 만을 이용하도록 로직 교체.
3.  **수익률 미세 변경 현상:** `datetime.now()` 실행 시점의 시/분/초 차이로 인해 기준일 경계가 어긋나는 문제 -> `replace(hour=0, minute=0, second=0)` 으로 정규화.
4.  **윈도우 터미널 입력 글자수 제한(Truncated) 오류:** 긴 코드를 EOF 방식으로 쓸 때 끝부분이 잘려나가는(Syntax Error) 문제 -> Python 스크립트 기반 쓰기/수정으로 안전하게 패치 완료.
