## Problem Statement
단순 지표 시각화가 아닌, 실전 기관급(Institutional-grade) 퀀트 환경을 원함.
추가 시그널(RSI, MACD), 수수료/슬리피지, 캐시 스탬피드, 생존자 편향 등을 한 번에("Boil the Lake") 해결하고자 함.
기존의 단순 반복(for-loop) 기반 백테스트는 기능 추가 시 연산 부하를 견디지 못하므로 아키텍처 재설계가 필요.

## What Makes This Cool
"포인트 인 타임(Point-in-Time) 타임머신 디버거".
특정 과거(예: 2024년 4월)로 시간 슬라이더를 옮기면, 미래 데이터 참조 없이 당시에만 존재했던 유니버스와 당일의 지표, 호가창 슬리피지가 반영된 결과가 **1초 내에 렌더링**됨. DuckDB의 디스크 읽기 성능과 VectorBT의 C단위 벡터 연산 속도로, Streamlit 웹 위에서 헤비한 퀀트 시뮬레이션이 실시간으로 동작함.

## Constraints
- 시장: 한국 주식 시장
- 프레임워크: Streamlit, Python
- 병목 최소화: API 요청 횟수를 극단적으로 줄이고 로컬 I/O로 전환
- 언어: 문서 및 화면 UI는 한국어 우선 (CLAUDE.md 규칙)

## Premises
1. **의존성 순서:** 외부 API(FinanceDataReader 등)의 Rate Limit과 중복 호출(Cache Stampede)을 막기 위해 로컬 캐싱 파이프라인(Parquet) 구축이 최우선.
2. **저장 방식:** 파일럿 단계의 편의성과 속도를 위해 Parquet + DuckDB 조합 선택.
3. **연산 부하 대응:** 백테스트 연산은 Pandas 순회를 버리고 VectorBT 기반 벡터 연산으로 교체.

## Cross-Model Perspective
SECOND OPINION (Claude subagent):
1. **COOLEST version:** DuckDB와 Parquet, Streamlit을 활용한 Point-in-Time 타임머신 디버거. Look-ahead bias 없이 과거 시점을 재구성.
2. **Key insight:** "Boil the Lake" (한 번에 모든 걸 다 만들겠다) - 무결점 엔진을 하드코어하게 설계하는 것에 흥미를 느낌.
3. **Tools:** VectorBT (병렬 연산 및 슬리피지/수수료) + DuckDB (초고속 로컬 파케이 쿼리). File-lock 기반 데이터 적재 파이프라인 직접 구축 필요.
4. **First step:** 캐시 스탬피드 방지용 Parquet + DuckDB 데이터 I/O 코어 파이프라인 구축.

## Approaches Considered
### Approach A: Python Multiprocessing + Parquet (실리주의)
  기존 로직 유지, multiprocessing으로 속도만 보완. VectorBT급 속도는 안 나오지만 익숙함.
### Approach B: DuckDB + VectorBT (이상적 아키텍처)
  데이터 I/O를 DuckDB로, 백테스트 엔진을 VectorBT로 교체. 커스텀 로직은 까다로우나 극강의 속도와 수수료/슬리피지 기본 지원.
### Approach C: Point-in-Time 파이프라인 올인
  UI/연산은 보류하고 데이터 파이프라인만 완성. 눈에 보이는 결과물이 늦게 나옴.

## Recommended Approach
**Approach B: DuckDB + VectorBT (이상적 아키텍처)**
초반의 학습 곡선과 `core_engine.py` 전면 재작성의 리스크를 감수하더라도, 한 번 구축하면 병렬 처리와 고급 기능(수수료, RSI 등)을 무료로 얻게 되며, 장기적으로 가장 튼튼한 무기가 됨.

## Open Questions
- VectorBT에서 한국 주식 시장의 상하한가 룰이나 특수한 슬리피지 로직을 완벽하게 커스텀할 수 있는가?
- 상장폐지 종목 데이터를 어디서 안정적으로 소싱하여 Parquet에 채워넣을 것인가?

## Success Criteria
1. 백엔드에서 `filelock` 등을 통해 다중 API 호출 없이 Parquet 파일로 일봉 데이터가 안전하게 적재됨.
2. `core_engine.py`의 핵심 연산이 VectorBT로 교체되어, 복수 파라미터(RSI, MACD, 이평선) 조합의 테스트가 1초 내에 완료됨.
3. 수수료 및 슬리피지 파라미터가 적용된 백테스트 결과가 UI에 표시됨.

## Next Steps
1. **기술 검증(POC):** VectorBT에서 한국 주식의 상하한가 룰(±30%)을 Numba 커스텀 함수로 구현했을 때 속도 저하가 없는지, 그리고 KRX/FDR에서 상장폐지 종목 데이터를 소싱할 수 있는지 확인.
2. `vectorbt`, `duckdb`, `pyarrow` 라이브러리 설치.
3. `core_engine.py` 내의 주가 데이터 수집 로직을 **Parquet 파일 저장 및 DuckDB 쿼리 로직**으로 변경 (캐시 스탬피드 해결).
4. 이동평균선 로직을 VectorBT의 `vbt.MA.run` 형식으로 교체하고, RSI/MACD 시그널 지표 추가.

## What I noticed about how you think
