## Problem Statement
기존 HTS/MTS에 의존하지 않고, 개인화된 백테스트와 실시간 자동 매매, 실시간 모니터링이 모두 가능한 독립적인 투자 환경을 구축하고자 함. 그 첫 마일스톤으로 "현재 우량주 중 거래대금 상위 10개 종목에 대한 2년치 백테스트 결과를 웹 대시보드로 실시간 확인"하는 것을 목표로 함.

## What Makes This Cool
단순히 터미널에 텍스트를 출력하는 스크립트가 아니라, **Streamlit**을 이용해 웹 브라우저 상에서 동작하는 인터랙티브 대시보드를 구축. 이동평균선 일수(5일, 20일 등)를 슬라이더로 조절하면 백테스트 결과 차트와 수익률이 즉각적으로 변하는 살아있는 앱 형태가 됨. 

## Constraints
- 시장: 한국 주식 시장 (코스피/코스닥)
- 개발 언어 및 도구: Python, pandas (데이터 처리), pykrx 또는 FinanceDataReader (데이터 수집)
- 무거운 기존 퀀트 프레임워크(Backtrader 등) 전체를 도입하는 것은 피하되, 빠른 벡터 연산이 가능한 `vectorbt`의 차트/통계 기능만 선택적으로 활용하여 구현 복잡도를 낮춤.

## Premises
1. **대상 시장:** 한국 주식 시장(코스피/코스닥)의 KOSPI 200 및 KOSDAQ 150 지수 편입 종목을 유니버스로 삼는다. (동전주/테마주 배제) 최근 거래일 기준으로 거래대금 상위 10개 종목을 추출하고 2년 치 일봉 데이터를 가져온다. (api limit 방지를 위해 1회 조회 후 `@st.cache_data`로 로컬 메모리에 캐싱)
2. **테스트 전략:** 단기/장기 이동평균선 교차 전략(골든크로스 매수, 데드크로스 매도)을 베이스라인으로 삼으며, 슬라이더를 통해 일수를 동적 조절한다.
3. **사용 도구:** 시그널 계산은 `pandas`로 직접 통제하고, 시각화와 웹 UI는 `Streamlit`과 `vectorbt`로 구축한다. 거래세(0.2%)와 수수료를 백테스트 로직에 필수로 반영한다.

## Approaches Considered
### Approach A: KOSPI 200 / KOSDAQ 150 풀 사용 (The Minimal Viable) - **[선택됨]**
  Summary: 거래소에서 공식적으로 제공하는 대표 지수(우량주 모음)에 편입된 종목 리스트를 가져와, 그 안에서 거래대금 상위 10개를 뽑습니다.
  Effort:  Small
  Risk:    Low
  Pros:    - 코드가 짧고, 시장에서 검증된 우량주만 필터링됨
  Cons:    - 지수에 편입되지 않은 핫한 신규 상장주 등은 놓칠 수 있음

### Approach B: 매일 시가총액 상위 100개를 동적 스크래핑 (The Ideal Architecture)
  Summary: 매일 전 종목의 시가총액을 계산하여 상위 100개를 추리고, 그 안에서 거래대금 상위 10개를 뽑습니다.
  Effort:  Medium
  Risk:    Low
  Pros:    - 지수 편입 여부와 무관하게 현재 가장 덩치가 큰 기업들을 정확히 타겟팅
  Cons:    - 연산량이 늘어나고, 로딩 시간이 길어질 수 있음

## Recommended Approach
**APPROACH A (KOSPI 200 / KOSDAQ 150 풀 사용)** 채택.
거래량 상위 필터의 함정(소형 테마주, 동전주 편입)을 회피하기 위해, 검증된 지수 편입 종목을 유니버스로 삼고 '거래대금'을 기준으로 필터링하여 백테스트의 신뢰도를 높임.

## Success Criteria
1. 코드를 실행하면 로컬 웹 브라우저에 대시보드가 뜬다 (`streamlit run app.py`).
2. 대시보드 진입 시 KOSPI 200 / KOSDAQ 150 종목 중 최근 거래일 거래대금 상위 10개 종목 리스트가 1회 API 호출(`@st.cache_data`)을 통해 불러와진다.
3. 종목을 선택하고 슬라이더로 이평선 값(예: 단기 5, 장기 20)을 조절하면, 과거 2년 치 데이터 기반의 수수료/세금이 반영된 백테스트 수익률 차트와 승률/MDD가 화면에 즉각 출력된다.

## Distribution Plan
로컬 환경 구동. 향후 Github에 올리고 Streamlit Community Cloud를 통해 무료로 웹 호스팅하여 개인 도메인으로 접속 가능하도록 확장.

## Next Steps
1. **Core 엔진 구현:** `pykrx` API를 이용해 KOSPI 200 / KOSDAQ 150 구성종목 티커를 수집하고, 해당 유니버스 내에서 거래대금 상위 10개 종목을 추출하는 함수 작성 (반드시 `@st.cache_data` 적용).
2. **시그널 처리:** `pandas`로 동적 이평선 시그널(True/False) 생성 및 거래세/수수료 반영 코드 작성.
3. **UI 씌우기:** `streamlit`을 이용해 종목 선택 드롭다운, 단기/장기 이평선 조절 슬라이더, 그리고 백테스트 차트를 출력하는 웹 페이지 구성.

## The Assignment
이번 주말의 단 하나 목표: **화면에 Streamlit 버튼과 빈 차트라도 띄우는 것.** 
새로운 거래대금 기준 종목 추출 로직을 `app.py`에 반영하고, `streamlit run app.py`가 에러 없이 돌아가게 만드는 것이 첫 번째 숙제입니다.
