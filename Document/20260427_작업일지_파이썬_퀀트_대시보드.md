# 작업 일지: 파이썬 퀀트 투자 대시보드 (Data Pipeline & VectorBT 전환)

**날짜:** 2026년 04월 27일
**프로젝트:** SthocWin (파이썬 퀀트 투자 대시보드)
**목표:** Cache Stampede 방지, VectorBT 연산 속도 확보, 그리고 안정적인 아키텍처 구축

## 1. 오늘 작업 요약
*   **새로운 데이터 I/O 파이프라인(`data_pipeline.py`) 구축:**
    *   **Cache Stampede 방지:** 다중 사용자 접속 시 발생하는 동일 데이터 중복 요청을 방지하기 위해 `filelock` 도입. Reader-Writer 락 분리 기법을 적용하여 쓰는 동안에만 락을 걸고 읽기 작업은 즉시 처리되도록 최적화.
    *   **Parquet + DuckDB 캐싱:** `FinanceDataReader`로 수집한 일봉 데이터 및 메타데이터(KRX 주식 목록)를 `.parquet` 형식으로 `data/` 폴더에 캐싱. 캐시 히트(Hit) 시 0.02초 만에 초고속으로 데이터를 로드할 수 있도록 DuckDB 인메모리 쿼리 적용.
    *   **안전성(Atomic Rename) 확보:** 다운로드 중 인터넷 끊김 등으로 인한 파일 오염(Corrupt)을 방지하고자, `.tmp` 임시 파일에 먼저 저장한 뒤 완료되면 `os.replace`로 확장자를 변경하는 원자적 쓰기(Atomic Write) 로직 적용.
    *   **병렬 다운로드 (ThreadPoolExecutor):** 최초 1회 데이터 수집(Cache Miss) 시 대기 시간을 1~2분에서 10초 내외로 단축하기 위해 10개 종목을 병렬로 다운로드하도록 구현.

*   **백테스트 연산 엔진 전면 교체 (`core_engine.py`):**
    *   기존의 느린 Pandas for-loop 순회 방식(단순 반복문)을 버리고, 초고속 벡터 연산을 지원하는 **`VectorBT` 프레임워크** 도입.
    *   개별 종목 데이터를 하나의 통합 DataFrame으로 합친 뒤(`concat`, `ffill()`, `dropna()`), `vbt.MA.run()`을 사용해 한 번의 행렬 연산으로 다중 파라미터 백테스트 수행.
    *   신규 상장 등 종가(Close) 데이터가 없거나 극히 적은 종목이 섞여 있을 때, 전체 연산이 마비되지 않도록 사전에 방어하는 예외 처리 로직(Fail Gracefully) 추가.
    *   `MultiIndex` 형태의 VectorBT 결과를 UI 렌더링에 적합한 딕셔너리로 언패킹하는 과정에서 발생했던 `KeyError` 이슈 해결.

*   **UI 컴포넌트(`app.py`) 연동 및 최적화:**
    *   대시보드 하단 상세 차트용 데이터를 기존 방식이 아닌 새 파이프라인(`fetch_stock_data_parallel`)을 통하도록 수정하여 통일성(Single Source of Truth) 확보.
    *   Streamlit의 향후 버전에서 지원 중단될 예정인 `use_container_width` 관련 Deprecation Warning 로그들을 모두 분석하여 `width` 옵션 또는 삭제 처리로 깔끔하게 수정 완료.
